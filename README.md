## Структура репозитория

```
OPBD/
├── ПРО/                                        # Предметная область (чемпионат)
├── ERДиаграммы/                                # ER-диаграммы в нотации Crow Foot
├── Спецификация/                               # Спецификация ПРО
├── Нормализация/                               # Нормализация (вариант 22)
├── SQL Запросы/                                # SQL-запросы (MySQL и PostgreSQL)
│   ├── Postgre/                                # PostgreSQL-запросы 1–50 + дампы БД
│   ├── Запрос детали/                          # Запросы по заданию «Детали»
│   └── ПРО запросы/                            # Запросы к базе данных ПРО
├── Запросы По индивидуальному заданию/         # Инд. задание (вариант 25)
├── Детали/                                     # Задание «Детали» (вариант VI/12)
│   └── image/                                  # Скриншоты выполнения запросов
├── Тригеры по индивидуальному заданию/         # Триггеры на MySQL и PostgreSQL
│   ├── запросы/                                # SQL-скрипты триггеров
│   └── images/                                 # Скриншоты выполнения
├── Индивидуальное задание. Представления(Views)/ # Views на MySQL и PostgreSQL
│   └── Изображения/                            # Скриншоты представлений
├── Процедуры/                                  # Процедуры и функции (вариант 25)
│   └── Изображения/                            # Скриншоты выполнения процедур
├── Описание Приложений/                        # Отчёт по Python-приложениям
│   └── images/                                 # Скриншоты интерфейсов
├── Приложения/                                 # Исходный код приложений
│   ├── QtImportTwoTables/                      # Python PyQt6 (5 скриптов)
│   └── testingQt/                              # Тестовый PyQt6-скрипт
├── Курсовая/                                   # Курсовая работа (ветклиника)
│   ├── project/                                # Go-бэкенд (API-сервер)
│   ├── fromt/                                  # HTML-клиент
│   ├── markdown файлы с описанием работы/      # ПРО, структура БД, роли и т.д.
│   └── pdfфайлы с описанием работы/            # PDF-версии описаний
├── Подготовка к Демо Экзамену/                 # Материалы к демо-экзамену
│   ├── Images/                                 # Изображения для экзамена
│   ├── imports/                                # Excel-файлы для импорта данных
│   └── pdf/                                    # Официальные задания (PDF)
├── Практика/                                   # Папка для практических заданий
├── Детали без нормализаций.md                  # Ссылка на Excel без нормализаций
└── Замена задания.md                           # Замена процедуры удаления
```

### Детали

**Путь:** [`Детали/`](https://github.com/YaeKitsune13/OPBD/tree/main/%D0%94%D0%B5%D1%82%D0%B0%D0%BB%D0%B8)

| Таблица | Описание |
| :--- | :--- |
| `s` (suppliers) | Поставщики (имя, рейтинг, город) |
| `p` (parts) | Детали (название, цвет, вес, город изготовления) |
| `j` (projects) | Продукты/проекты (название, город) |
| `spj` | Связная таблица поставок (поставщик → деталь → продукт, кол-во, дата) |

| Запрос 2 | Вставить нового поставщика с заданными параметрами |
| Запрос 3 | Вывести поставщиков, поставивших указанную деталь в заданный период |
| Запрос 4 | Удалить поставщика с наименьшим числом поставок |

| Файл | Содержимое |

**Путь:** [`Тригеры по индивидуальному заданию/`](https://github.com/YaeKitsune13/OPBD/tree/main/%D0%A2%D1%80%D0%B8%D0%B3%D0%B5%D1%80%D1%8B%20%D0%BF%D0%BE%20%D0%B8%D0%BD%D0%B4%D0%B8%D0%B2%D0%B8%D0%B4%D1%83%D0%B0%D0%BB%D1%8C%D0%BD%D0%BE%D0%BC%D1%83%20%D0%B7%D0%B0%D0%B4%D0%B0%D0%BD%D0%B8%D1%8E)

Триггеры к базе данных страховой компании **`insurance`** (вариант 25). Реализованы на **MySQL** и **PostgreSQL**. Документация в [`ТригерыMySql.md`](./Тригеры%20по%20индивидуальному%20заданию/ТригерыMySql.md).

| № | Триггер | Событие | Описание |
| :-: | :--- | :--- | :--- |
| 1 | `db_insert_policyholder_date_checker` | `BEFORE INSERT` | Проверяет, что дата начала договора не позже даты окончания; при успехе — логирует |
| 2 | `db_before_update_policyholder_date_checker` | `BEFORE UPDATE` | Запрещает некорректное обновление дат; логирует продление или сокращение договора |
| 3 | `db_insert_policyholders_calculate_sum` | `BEFORE INSERT` | Автоматически рассчитывает `policy_cost` = `premium_amount × количество дней` |
| 4 | `db_update_employers_if_insert_policyholder` | `AFTER INSERT` | Увеличивает счётчик `count_policyholders` у сотрудника после добавления нового договора |
| 5 | `db_remove_set_log` | `BEFORE DELETE` | Логирует расторжение договора с указанием его длительности |

| Файл | Содержимое |

### Приложения

**Путь:** [`Приложения/`](https://github.com/YaeKitsune13/OPBD/tree/main/%D0%9F%D1%80%D0%B8%D0%BB%D0%BE%D0%B6%D0%B5%D0%BD%D0%B8%D1%8F)

Исходный код клиентских приложений для работы с базами данных MySQL и PostgreSQL (БД `insurance`).

#### Подпапка `QtImportTwoTables/` — Python (PyQt6)

Пять скриптов, демонстрирующих разные подходы к отображению и фильтрации данных:

| Скрипт | Описание |
| :--- | :--- |
| `first.py` | Вывод данных в двух таблицах: список сотрудников и связанные страховые случаи (мастер-детали) |
| `second.py` | Выбор сотрудника через выпадающий список `QComboBox` вместо таблицы |
| `third.py` | Двухступенчатая фильтрация: сотрудник → страхователь → страховые случаи |
| `fourth.py` | Карточный интерфейс (`QFrame` + `QScrollArea`) вместо таблицы |
| `fiveth.py` | Детальные карточки с техническими полями и стилизацией через `setStyleSheet` |
| `addremoveupdate.py` | Операции добавления, удаления и обновления записей |

Все скрипты подключаются к двум СУБД одновременно (MySQL через `QMARIADB`, PostgreSQL через `QPSQL`).

### Курсовая

**Путь:** [`Курсовая/`](https://github.com/YaeKitsune13/OPBD/tree/main/%D0%9A%D1%83%D1%80%D1%81%D0%BE%D0%B2%D0%B0%D1%8F)

Курсовой проект — **информационная система ветеринарной клиники**. Включает серверную часть (API на Go) и клиентское веб-приложение (HTML).

#### Предметная область

Система предназначена для учёта посещений и оказания медицинских услуг в ветеринарной клинике. Поддерживает три роли пользователей: **врачи**, **владельцы животных**, **администрация клиники**.

#### Структура базы данных

| Таблица | Описание |
| :--- | :--- |
| `owners` | Владельцы животных (ФИО, адрес, телефон, email) |
| `pets` | Питомцы (кличка, вид, порода, дата рождения, вес) |
| `weight_history` | История изменения веса питомцев |
| `doctors` | Врачи (ФИО, специализация, телефон) |
| `services` | Справочник услуг (название, описание, стоимость) |
| `medications` | Справочник медикаментов (название, описание, цена за единицу) |
| `appointments` | Записи на приём (питомец, врач, дата/время, статус) |
| `appointment_status_log` | История изменения статусов заявок |
| `visits` | Медицинские карты визитов (анамнез, диагноз, итоговая стоимость) |
| `visit_services` | Назначенные услуги в рамках визита |
| `visit_medications` | Назначенные медикаменты в рамках визита |

#### Структура папки

| Путь | Содержимое |
| :--- | :--- |
| `project/main.go` | Точка входа Go-сервера (API) |
| `project/go.mod` | Модульный файл Go |
| `fromt/vetclinic.html` | HTML-клиент ветеринарной клиники |
| `ReadMe.md` | Инструкции по форматированию пояснительной записки |
| `markdown файлы с описанием работы/` | Описание ПРО, структуры БД, ролей, ограничений, входных/выходных данных |
| `pdfфайлы с описанием работы/` | PDF-версия описания курсовой работы |

| **Docker** | Запуск СУБД в контейнерах при разработке |
| **Visio** | Построение ER-диаграмм |
