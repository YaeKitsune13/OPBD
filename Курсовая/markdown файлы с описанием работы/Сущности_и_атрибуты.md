# Определение сущностей и атрибутов информационной системы

## 1. Сущность: Владелец (Owner)
*Представляет собой профиль клиента клиники.*
- **Фамилия, Имя, Отчество:** Обязательные атрибуты. В базе и интерфейсе разделены для корректного формирования печатных форм (Иванов И.И.).
- **Адрес проживания:** Текстовое поле (до 255 симв.). Необязательно для первичной регистрации, но необходимо для формирования официальных договоров.
- **Контактный телефон:** Уникальный атрибут. Валидация по маске `+7 (XXX) XXX-XX-XX`. Обязательное поле для связи.
- **Email:** Уникальный идентификатор пользователя (логин). Проверка на соответствие формату адреса электронной почты.
- **Пароль:** Хранится в зашифрованном виде. Требование — не менее 8 символов.

## 2. Сущность: Питомец (Pet)
*Объект медицинского учета.*
- **Кличка:** Строго текстовый формат (кириллица/латиница). Длина от 1 до 30 символов. Цифры заблокированы на уровне интерфейса.
- **Вид животного:** Атрибут типа Enum. Ограничен списком: Кошка, Собака, Кролик, Птица, Грызун, Другое.
- **Порода:** Текстовое поле. Если порода неизвестна, указывается «Беспородная».
- **Дата рождения:** Используется для автоматического вычисления возраста (отображается в тегах, например, «5 лет»). Не может быть позже текущей даты.
- **Вес:** Положительное число (Decimal). Ограничение: от 0.1 до 300.0 кг. Служит основой для построения графика динамики.
- **Фото:** Ссылка на файл изображения (JPG/PNG). Ограничение по размеру — до 5 МБ.

## 3. Сущность: Врач (Doctor)
*Сотрудник клиники, осуществляющий прием.*
- **ФИО:** Полное имя сотрудника.
- **Специализация:** Строго фиксированный список: Терапевт, Хирург, Кардиолог, Невролог, Дерматолог.
- **Рабочий график:** Совокупность доступных временных слотов для записи (шаг — 30 минут, интервал 09:00–18:00).

## 4. Сущность: Медикамент (Medication)
*Товарная позиция аптечного склада.*
- **Название и Описание:** Текстовые поля.
- **Цена за единицу:** Денежный формат. Не может быть меньше 1 рубля.
- **Срок годности:** Поле типа DATE. Используется для автоматической фильтрации просроченных препаратов в модуле «Администрация».

## 5. Сущность: Визит (Visit)
*Запись в Журнале здоровья, результат приема.*
- **Анамнез:** Обязательный текстовый блок. Длина от 10 до 2000 символов (жалобы, состояние, предыстория болезни).
- **Диагноз:** Обязательное поле. Краткая формулировка выявленного заболевания.
- **Назначения:** Совокупный список (связь «многие-ко-многим») с услугами и медикаментами.
- **Итоговая сумма:** Вычисляемый атрибут. Сумма стоимостей всех добавленных в визит назначений.
