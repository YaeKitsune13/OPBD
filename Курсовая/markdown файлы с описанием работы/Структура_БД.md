# Структура базы данных и описание DTO-моделей

## 1. Реляционная структура базы данных

### Схема связей

```
owners ──< pets ──< appointments >── doctors
                         │
                       visits ──< visit_prescriptions >── services
                         │                          └──< medications
                         └──> weight_history
```

### Таблица 1.1: owners (Владельцы / Клиенты)
| Поле | Тип данных | Ограничения | Описание |
| :--- | :--- | :--- | :--- |
| owner_id | INTEGER | PK, Auto Increment | Уникальный код владельца |
| last_name | VARCHAR(50) | NOT NULL | Фамилия |
| first_name | VARCHAR(50) | NOT NULL | Имя |
| middle_name | VARCHAR(50) | NULL | Отчество |
| address | VARCHAR(255) | NULL | Домашний адрес |
| phone | VARCHAR(15) | UNIQUE, NOT NULL | Контактный телефон |
| email | VARCHAR(100) | UNIQUE, NOT NULL | Логин (почта) |
| password_hash | VARCHAR(255) | NOT NULL | Хэш пароля |

### Таблица 1.2: pets (Питомцы)
| Поле | Тип данных | Ограничения | Описание |
| :--- | :--- | :--- | :--- |
| pet_id | INTEGER | PK, Auto Increment | Код питомца |
| owner_id | INTEGER | FK (owners.owner_id) | Владелец |
| nickname | VARCHAR(30) | NOT NULL | Кличка (только буквы) |
| species | VARCHAR(50) | NOT NULL | Вид (Кошка, Собака и др.) |
| breed | VARCHAR(100) | NULL | Порода |
| birth_date | DATE | NOT NULL | Дата рождения |
| photo | VARCHAR(255) | NULL | Путь к файлу изображения |
| current_weight | DECIMAL(5,2) | CHECK (0.1-300) | Последний зафиксированный вес |

### Таблица 1.3: doctors (Врачи)
| Поле | Тип данных | Ограничения | Описание |
| :--- | :--- | :--- | :--- |
| doctor_id | INTEGER | PK, Auto Increment | Уникальный код врача |
| last_name | VARCHAR(50) | NOT NULL | Фамилия |
| first_name | VARCHAR(50) | NOT NULL | Имя |
| middle_name | VARCHAR(50) | NULL | Отчество |
| specialty | VARCHAR(50) | NOT NULL | Специализация (Терапевт, Хирург, Кардиолог, Невролог, Дерматолог) |
| phone | VARCHAR(15) | NULL | Контактный телефон |
| email | VARCHAR(100) | UNIQUE, NOT NULL | Логин врача |
| password_hash | VARCHAR(255) | NOT NULL | Хэш пароля |

### Таблица 1.4: appointments (Заявки на прием)
| Поле | Тип данных | Ограничения | Описание |
| :--- | :--- | :--- | :--- |
| appointment_id | INTEGER | PK, Auto Increment | Номер заявки |
| pet_id | INTEGER | FK (pets.pet_id) | Кто записан |
| doctor_id | INTEGER | FK (doctors.doctor_id) | К какому врачу |
| scheduled_at | DATETIME | NOT NULL | Плановая дата и время |
| comment | TEXT | NULL | Комментарий/жалоба клиента |
| status | ENUM | 'waiting','confirmed','rejected' | Статус записи |
| created_at | DATETIME | DEFAULT NOW() | Дата создания заявки |

### Таблица 1.5: visits (Медицинские карты приема)
| Поле | Тип данных | Ограничения | Описание |
| :--- | :--- | :--- | :--- |
| visit_id | INTEGER | PK, Auto Increment | Код медкарты |
| appointment_id | INTEGER | FK (appointments.appointment_id) | Ссылка на запись |
| anamnesis | TEXT | NOT NULL, CHECK (LEN 10-2000) | Описание состояния / жалобы |
| diagnosis | VARCHAR(500) | NOT NULL | Поставленный диагноз |
| total_cost | DECIMAL(10,2) | NOT NULL, DEFAULT 0 | Итоговая сумма за прием (вычисляется автоматически) |
| visit_date | DATETIME | DEFAULT NOW() | Фактическая дата проведения приема |

### Таблица 1.6: services (Справочник услуг)
| Поле | Тип данных | Ограничения | Описание |
| :--- | :--- | :--- | :--- |
| service_id | INTEGER | PK, Auto Increment | Код услуги |
| code | VARCHAR(10) | UNIQUE, NOT NULL | Внутренний артикул (001, 002 …) |
| name | VARCHAR(150) | NOT NULL | Наименование услуги |
| description | TEXT | NULL | Описание процедуры |
| price | DECIMAL(10,2) | NOT NULL, CHECK (>0) | Стоимость |

### Таблица 1.7: medications (Справочник медикаментов)
| Поле | Тип данных | Ограничения | Описание |
| :--- | :--- | :--- | :--- |
| medication_id | INTEGER | PK, Auto Increment | Код препарата |
| code | VARCHAR(10) | UNIQUE, NOT NULL | Артикул (M001, M002 …) |
| name | VARCHAR(150) | NOT NULL | Название |
| description | TEXT | NULL | Описание / показания |
| price_per_unit | DECIMAL(10,2) | NOT NULL, CHECK (>0) | Цена за 1 ед. |
| expiry_date | DATE | NOT NULL | Срок годности |
| status | ENUM | 'ok', 'expired' | Вычисляемый статус годности |

### Таблица 1.8: visit_prescriptions (Назначения визита)
Связывает визит со списком оказанных услуг и медикаментов (связь «многие-ко-многим»).

| Поле | Тип данных | Ограничения | Описание |
| :--- | :--- | :--- | :--- |
| prescription_id | INTEGER | PK, Auto Increment | Код строки назначения |
| visit_id | INTEGER | FK (visits.visit_id) | Ссылка на медкарту |
| item_type | ENUM | 'service', 'medication' | Тип позиции |
| service_id | INTEGER | FK (services.service_id), NULL | Услуга (если item_type = 'service') |
| medication_id | INTEGER | FK (medications.medication_id), NULL | Препарат (если item_type = 'medication') |
| quantity | INTEGER | NOT NULL, DEFAULT 1 | Количество единиц |
| unit_price | DECIMAL(10,2) | NOT NULL | Цена на момент назначения |

### Таблица 1.9: weight_history (История весов)
| Поле | Тип данных | Ограничения | Описание |
| :--- | :--- | :--- | :--- |
| record_id | INTEGER | PK, Auto Increment | Код записи |
| pet_id | INTEGER | FK (pets.pet_id) | Питомец |
| visit_id | INTEGER | FK (visits.visit_id), NULL | Визит, на котором взвешен (NULL для ручных записей) |
| weight_kg | DECIMAL(5,2) | NOT NULL, CHECK (0.1-300) | Зафиксированный вес |
| measured_at | DATETIME | NOT NULL | Дата и время измерения |
| doctor_id | INTEGER | FK (doctors.doctor_id), NULL | Врач, взвесивший питомца |

## 2. Описание DTO (Data Transfer Objects) для вывода в UI

DTO используются для передачи агрегированных данных из базы в интерфейс пользователя.

### 2.1 DashboardDTO (Экран "Обзор" — `DashboardPage.vue`)
Используется для вывода сводной информации на главной странице клиента.
- `clientName` (String): Имя для приветствия ("Иван").
- `petsCount` (Integer): Общее число питомцев (например, 3).
- `pendingApps` (Integer): Количество заявок в статусе "Ожидание".
- `totalVisits` (Integer): Суммарное количество визитов за всё время.
- `nextAppointment` (Object): `{ date, time, petName, doctorName }` — данные ближайшего приёма.
- `recentAppointments` (Array): Список последних 3–5 записей для виджета.
- `petsShort` (Array): Краткий список питомцев `{ name, species, breed, weight }` для виджета.

### 2.2 PetCardDTO (Экран "Мои питомцы" — `PetsPage.vue`)
Данные одной карточки питомца.
- `petId` (Integer): Идентификатор.
- `name` (String): Кличка.
- `species` (String): Вид.
- `breed` (String): Порода.
- `dob` (String): Дата рождения.
- `weight` (Decimal): Текущий вес в кг.
- `avatar` (String): Эмодзи-аватар, выбирается по виду (🐱, 🐶, 🐇).

### 2.3 AppointmentRowDTO (Экран "Записи" — `AppointmentsPage.vue`)
Данные одной строки таблицы заявок.
- `appointmentId` (Integer): Номер #ID.
- `petLabel` (String): Эмодзи + кличка.
- `doctorName` (String): ФИО врача.
- `specialty` (String): Специализация.
- `scheduledDate` (String): Дата.
- `scheduledTime` (String): Время.
- `status` (Enum): 'waiting' | 'confirmed' | 'rejected'.

### 2.4 HealthJournalDTO (Экран "Журнал здоровья" — `HistoryPage.vue`)
Данные одной карточки визита.
- `visitId` (Integer): Идентификатор.
- `date` (String): Дата визита.
- `time` (String): Время.
- `doctor` (String): ФИО врача.
- `diagnosis` (String): Диагноз с иконкой.
- `details` (String): Список назначений.
- `analysis` (String | null): Результаты анализов.
- `recommendations` (String | null): Рекомендации.
- `price` (String): Итоговая стоимость.

### 2.5 WeightPointDTO (Экран "Динамика веса" — `WeightPage.vue`)
Одна точка на графике веса.
- `label` (String): Подпись по оси X (месяц, например "апр").
- `value` (Decimal): Значение веса в кг.
- `date` (String): Полная дата измерения.
- `doctorName` (String): Врач, взвесивший питомца.

### 2.6 TodayScheduleDTO (Экран "Расписание" — `TodayPage.vue`)
Одна запись в расписании врача на день.
- `appointmentId` (Integer): Идентификатор.
- `time` (String): Время приёма.
- `petLabel` (String): Эмодзи + кличка.
- `ownerName` (String): ФИО владельца.
- `breed` (String): Порода.
- `reason` (String): Причина визита.
- `status` (Enum): 'waiting' | 'confirmed'.

### 2.7 ConductVisitDTO (Экран "Ведение приёма" — `ConductPage.vue`)
Данные текущего приёма.
- `selectedPet` (Object): `{ id, name, avatar, breed, owner }` — выбранный питомец.
- `anamnesis` (String): Текст жалоб.
- `diagnosis` (String): Введённый диагноз.
- `assignments` (Array): `[{ id, name, type, price, qty }]` — список назначений.
- `totalCost` (Integer): Вычисляемая сумма.

### 2.8 AnalyticsSummaryDTO (Экран "Аналитика" — `AnalyticsPage.vue`)
Сводные данные по KPI клиники.
- `monthlyVisits` (Integer): Визитов за месяц.
- `totalRevenue` (String): Суммарная выручка.
- `revenueChange` (String): Изменение к прошлому месяцу ("+12%").
- `activeClients` (Integer): Активных клиентов.
- `avgCheck` (Integer): Средний чек в рублях.
- `popularServices` (Array): `[{ name, count, revenue }]`.
- `doctorLoad` (Array): `[{ name, visitCount, loadStatus }]`.

### 2.9 RevenueReportDTO (Экран "Выручка" — `RevenuePage.vue`)
Детализация выручки за период.
- `periodTotal` (Integer): Итого за период.
- `servicesTotal` (Integer): За услуги.
- `dailyRows` (Array): `[{ date, visits, services, meds, total }]` — строки таблицы.

### 2.10 ServiceItemDTO (Экран "Услуги" — `ServicesPage.vue`)
Позиция справочника услуг.
- `id` (String): Артикул (001 …).
- `name` (String): Наименование.
- `desc` (String): Описание.
- `price` (Integer): Стоимость в рублях.

### 2.11 MedicationItemDTO (Экран "Медикаменты" — `MedsPage.vue`)
Позиция аптечного склада.
- `id` (String): Артикул (M001 …).
- `name` (String): Название.
- `desc` (String): Описание / показания.
- `price` (Integer): Цена за единицу.
- `expiry` (String): Срок годности (YYYY-MM).
- `status` (Enum): 'ok' | 'expired'.
