# Описание структуры базы данных

---

## Таблицы и связи

### owners — Владельцы

| Поле | Тип | Ключ | Описание |
| :--- | :--- | :--- | :--- |
| owner_id | INTEGER | PK | Уникальный идентификатор |
| last_name | VARCHAR(50) | — | Фамилия |
| first_name | VARCHAR(50) | — | Имя |
| middle_name | VARCHAR(50) | — | Отчество |
| address | VARCHAR(255) | — | Домашний адрес |
| phone | VARCHAR(12) | UNIQUE | Контактный телефон |
| email | VARCHAR(100) | UNIQUE | Email |
| password_hash | VARCHAR(255) | — | Хэш пароля |

---

### pets — Питомцы

| Поле | Тип | Ключ | Описание |
| :--- | :--- | :--- | :--- |
| pet_id | INTEGER | PK | Уникальный идентификатор |
| owner_id | INTEGER | FK → owners | Владелец |
| nickname | VARCHAR(30) | — | Кличка |
| species | VARCHAR(50) | — | Вид животного |
| breed | VARCHAR(50) | — | Порода |
| birth_date | DATE | — | Дата рождения |
| photo | VARCHAR(255) | — | Путь к фото |
| current_weight | DECIMAL(5,2) | — | Текущий вес (кг) |

---

### weight_history — История веса

| Поле | Тип | Ключ | Описание |
| :--- | :--- | :--- | :--- |
| weight_id | INTEGER | PK | Уникальный идентификатор |
| pet_id | INTEGER | FK → pets | Питомец |
| weight | DECIMAL(5,2) | — | Вес (кг) |
| measured_at | DATE | — | Дата измерения |

---

### doctors — Врачи

| Поле | Тип | Ключ | Описание |
| :--- | :--- | :--- | :--- |
| doctor_id | INTEGER | PK | Уникальный идентификатор |
| last_name | VARCHAR(50) | — | Фамилия |
| first_name | VARCHAR(50) | — | Имя |
| middle_name | VARCHAR(50) | — | Отчество |
| specialization | VARCHAR(100) | — | Специализация |
| phone | VARCHAR(12) | UNIQUE | Телефон |

---

### services — Услуги

| Поле | Тип | Ключ | Описание |
| :--- | :--- | :--- | :--- |
| service_id | INTEGER | PK | Уникальный идентификатор |
| name | VARCHAR(100) | — | Наименование |
| description | TEXT | — | Описание процедуры |
| price | DECIMAL(10,2) | — | Стоимость |

---

### medications — Медикаменты

| Поле | Тип | Ключ | Описание |
| :--- | :--- | :--- | :--- |
| medication_id | INTEGER | PK | Уникальный идентификатор |
| name | VARCHAR(100) | — | Название |
| description | TEXT | — | Описание |
| price_per_unit | DECIMAL(10,2) | — | Цена за единицу |

---

### appointments — Записи на приём

| Поле | Тип | Ключ | Описание |
| :--- | :--- | :--- | :--- |
| appointment_id | INTEGER | PK | Уникальный идентификатор |
| pet_id | INTEGER | FK → pets | Питомец |
| doctor_id | INTEGER | FK → doctors | Врач |
| scheduled_at | DATETIME | — | Дата и время приёма |
| status | ENUM('waiting','confirmed','rejected') | — | Статус заявки |

---

### appointment_status_log — История статусов

| Поле | Тип | Ключ | Описание |
| :--- | :--- | :--- | :--- |
| log_id | INTEGER | PK | Уникальный идентификатор |
| appointment_id | INTEGER | FK → appointments | Запись на приём |
| status | ENUM('waiting','confirmed','rejected') | — | Новый статус |
| changed_by | VARCHAR(100) | — | Автор изменения |
| changed_at | DATETIME | — | Время изменения |

---

### visits — Медицинские карты визитов

| Поле | Тип | Ключ | Описание |
| :--- | :--- | :--- | :--- |
| visit_id | INTEGER | PK | Уникальный идентификатор |
| appointment_id | INTEGER | FK → appointments, UNIQUE | Запись на приём |
| anamnesis | TEXT | — | Анамнез |
| diagnosis | VARCHAR(500) | — | Диагноз |
| total_cost | DECIMAL(10,2) | — | Итоговая стоимость |

---

### visit_services — Назначенные услуги

| Поле | Тип | Ключ | Описание |
| :--- | :--- | :--- | :--- |
| visit_service_id | INTEGER | PK | Уникальный идентификатор |
| visit_id | INTEGER | FK → visits | Визит |
| service_id | INTEGER | FK → services | Услуга |
| quantity | INTEGER | — | Количество |

---

### visit_medications — Назначенные медикаменты

| Поле | Тип | Ключ | Описание |
| :--- | :--- | :--- | :--- |
| visit_medication_id | INTEGER | PK | Уникальный идентификатор |
| visit_id | INTEGER | FK → visits | Визит |
| medication_id | INTEGER | FK → medications | Медикамент |
| quantity | DECIMAL(6,2) | — | Количество (доз/упаковок) |

---

## Схема связей

    owners ──< pets ──< weight_history
                  |
                  └──< appointments ──< appointment_status_log
                             |
                             └──< visits ──< visit_services ──> services
                                       |
                                       └──< visit_medications ──> medications
    doctors ──< appointments
