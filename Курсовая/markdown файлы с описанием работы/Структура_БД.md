# Описание структуры базы данных

В качестве СУБД используется **MySQL**. База данных содержит 9 таблиц.

---

**Таблица owners** (Владельцы)

| Содержание поля | Имя поля | Тип, длина | Примечания |
| :--- | :--- | :--- | :--- |
| Идентификатор владельца | owner_id | INT | первичный ключ, Auto Increment |
| Фамилия | last_name | VARCHAR(50) | обязательное поле |
| Имя | first_name | VARCHAR(50) | обязательное поле |
| Отчество | middle_name | VARCHAR(50) | необязательное поле |
| Адрес проживания | address | VARCHAR(255) | необязательное поле |
| Контактный телефон | phone | VARCHAR(20) | обязательное, уникальное поле |
| Email | email | VARCHAR(255) | обязательное, уникальное поле, логин |
| Хэш пароля | password_hash | VARCHAR(255) | обязательное поле |

---

**Таблица pets** (Питомцы)

| Содержание поля | Имя поля | Тип, длина | Примечания |
| :--- | :--- | :--- | :--- |
| Идентификатор питомца | pet_id | INT | первичный ключ, Auto Increment |
| Идентификатор владельца | owner_id | INT | внешний ключ (owners) |
| Кличка | nickname | VARCHAR(30) | обязательное поле, только буквы |
| Вид животного | species | ENUM | Кошка, Собака, Кролик, Другое |
| Порода | breed | VARCHAR(100) | необязательное поле |
| Дата рождения | birth_date | DATE | обязательное поле, не позже текущей даты |
| Текущий вес | current_weight | DECIMAL(5,2) | диапазон 0.1–300.0 кг |
| Фото | photo | VARCHAR(255) | ссылка на файл JPG/PNG, до 5 МБ |

---

**Таблица doctors** (Врачи)

| Содержание поля | Имя поля | Тип, длина | Примечания |
| :--- | :--- | :--- | :--- |
| Идентификатор врача | doctor_id | INT | первичный ключ, Auto Increment |
| Фамилия | last_name | VARCHAR(50) | обязательное поле |
| Имя | first_name | VARCHAR(50) | обязательное поле |
| Отчество | middle_name | VARCHAR(50) | необязательное поле |
| Специализация | specialty | ENUM | Терапевт, Хирург, Кардиолог, Невролог, Дерматолог |
| Email | email | VARCHAR(255) | обязательное, уникальное поле, логин |
| Хэш пароля | password_hash | VARCHAR(255) | обязательное поле |
| Контактный телефон | phone | VARCHAR(20) | необязательное поле |

---

**Таблица appointments** (Заявки на приём)

| Содержание поля | Имя поля | Тип, длина | Примечания |
| :--- | :--- | :--- | :--- |
| Идентификатор заявки | appointment_id | INT | первичный ключ, Auto Increment |
| Идентификатор питомца | pet_id | INT | внешний ключ (pets) |
| Идентификатор врача | doctor_id | INT | внешний ключ (doctors) |
| Дата и время приёма | scheduled_at | DATETIME | обязательное поле, только будущая дата |
| Комментарий | comment | VARCHAR(1000) | необязательное поле |
| Статус | status | ENUM | waiting, confirmed, rejected |
| Дата создания заявки | created_at | DATETIME | автоматически заполняется |

---

**Таблица visits** (Визиты / Медкарты)

| Содержание поля | Имя поля | Тип, длина | Примечания |
| :--- | :--- | :--- | :--- |
| Идентификатор визита | visit_id | INT | первичный ключ, Auto Increment |
| Идентификатор заявки | appointment_id | INT | внешний ключ (appointments) |
| Анамнез | anamnesis | TEXT | обязательное поле, 10–2000 символов |
| Диагноз | diagnosis | VARCHAR(255) | обязательное поле |
| Итоговая сумма | total_cost | DECIMAL(10,2) | вычисляется автоматически |
| Дата визита | visit_date | DATETIME | обязательное поле |

---

**Таблица services** (Услуги)

| Содержание поля | Имя поля | Тип, длина | Примечания |
| :--- | :--- | :--- | :--- |
| Идентификатор услуги | service_id | INT | первичный ключ, Auto Increment |
| Артикул | code | VARCHAR(10) | уникальное поле, генерируется автоматически |
| Наименование | name | VARCHAR(255) | обязательное поле |
| Описание | description | TEXT | необязательное поле |
| Стоимость | price | INT | обязательное поле, не менее 1 рубля |

---

**Таблица medications** (Медикаменты)

| Содержание поля | Имя поля | Тип, длина | Примечания |
| :--- | :--- | :--- | :--- |
| Идентификатор медикамента | medication_id | INT | первичный ключ, Auto Increment |
| Артикул | code | VARCHAR(10) | уникальное поле, генерируется автоматически |
| Название | name | VARCHAR(255) | обязательное поле |
| Описание | description | TEXT | необязательное поле |
| Цена за единицу | price_per_unit | DECIMAL(10,2) | обязательное поле, не менее 1 рубля |
| Срок годности | expiry_date | DATE | обязательное поле |
| Статус | status | ENUM | ok, expired — вычисляется автоматически |

---

**Таблица visit_prescriptions** (Назначения визита)

| Содержание поля | Имя поля | Тип, длина | Примечания |
| :--- | :--- | :--- | :--- |
| Идентификатор назначения | prescription_id | INT | первичный ключ, Auto Increment |
| Идентификатор визита | visit_id | INT | внешний ключ (visits) |
| Тип позиции | item_type | ENUM | service, medication |
| Идентификатор услуги | service_id | INT | внешний ключ (services), может быть NULL |
| Идентификатор медикамента | medication_id | INT | внешний ключ (medications), может быть NULL |
| Количество | quantity | INT | обязательное поле, не менее 1 |
| Цена на момент назначения | unit_price | DECIMAL(10,2) | фиксируется при сохранении |

---

**Таблица weight_history** (История весов)

| Содержание поля | Имя поля | Тип, длина | Примечания |
| :--- | :--- | :--- | :--- |
| Идентификатор записи | record_id | INT | первичный ключ, Auto Increment |
| Идентификатор питомца | pet_id | INT | внешний ключ (pets) |
| Идентификатор визита | visit_id | INT | внешний ключ (visits), может быть NULL |
| Вес | weight_kg | DECIMAL(5,2) | диапазон 0.1–300.0 кг |
| Дата измерения | measured_at | DATETIME | обязательное поле |
| Идентификатор врача | doctor_id | INT | внешний ключ (doctors) |
