# Ограничения на ввод данных

## 1. Профиль и Авторизация — `Auth.vue`
| Поле | Ограничение | Сообщение / Поведение в UI |
| :--- | :--- | :--- |
| **Email** | Формат адреса электронной почты (`@` + домен) | `alert('Введите почту и пароль')` при пустом поле |
| **Пароль** | Минимум 8 символов | Placeholder: «Минимум 8 символов» |
| **Имя (регистрация)** | Обязательное поле | `alert('Заполните обязательные поля')` |
| **Роль** | Только из набора: `client`, `doctor`, `admin` | Назначается автоматически по email; вручную не вводится |
| **Очистка пароля** | После успешного входа | `loginData.pass = ''` и `regData.pass = ''` — поля очищаются |

## 2. Картотека питомцев — `PetsPage.vue`
| Поле | Ограничение | Подсказка / Логика в UI |
| :--- | :--- | :--- |
| **Кличка** | Только буквы, 1–30 симв. | Placeholder: «Только буквы»; `alert` при пустом поле |
| **Вес (кг)** | Число от 0.1 до 300.0 | `type="number"`, `step="0.1"`, `min="0.1"`, `max="300"` |
| **Дата рождения** | Не может быть в будущем | `type="date"`; проверяется при сохранении |
| **Вид животного** | Фиксированный список | Кошка / Собака / Кролик / Другое; определяет avatar-эмодзи |
| **Удаление питомца** | Требует подтверждения | Открывает `BaseModal` с текстом «Это действие необратимо» |

## 3. Запись на приём — `BookPage.vue`
| Поле | Ограничение | Логика UI |
| :--- | :--- | :--- |
| **Специализация** | Обязательный выбор перед врачом | Пока не выбрана, `<select>` врача отключён (`:disabled="!selectedSpec"`) |
| **Врач** | Фильтрация по специализации | Список врачей обновляется через `computed availableDoctors` |
| **Дата приёма** | Не ранее следующего дня | `type="date"`, `min` = завтрашняя дата |
| **Питомец** | Выбор только из собственных | `<select>` заполнен только питомцами текущего клиента |

## 4. Ведение приёма — `ConductPage.vue`
| Поле | Ограничение | Логика валидации |
| :--- | :--- | :--- |
| **Питомец** | Обязательный выбор | `PetCombobox` с дефолтным первым питомцем |
| **Анамнез** | 10–2000 символов | Обязательное поле `<textarea>` |
| **Диагноз** | Непустое текстовое поле | Обязательное для сохранения медкарты |
| **Назначения** | Минимум 1 позиция | Кнопка «Сохранить прием» активна при наличии хотя бы одного назначения |
| **Удаление назначения** | Немедленное без подтверждения | Кнопка ✕ вызывает `removeItem(id)` |
| **Стоимость** | Вычисляется автоматически | `computed totalCost` = сумма `price * qty` по всем назначениям |

## 5. Управление услугами — `ServicesPage.vue`
| Поле | Ограничение | Логика UI |
| :--- | :--- | :--- |
| **Наименование** | Непустое поле | `alert('Пожалуйста, заполните название и цену')` |
| **Стоимость** | Число > 0 | `type="number"`, `min="1"`; отрицательные значения заблокированы |
| **ID (артикул)** | Генерируется автоматически | `String(services.length + 1).padStart(3, '0')` |
| **Удаление** | Требует подтверждения | Нативный `confirm('Удалить эту услугу из справочника?')` |

## 6. Управление медикаментами — `MedsPage.vue`
| Поле | Ограничение | Логика UI |
| :--- | :--- | :--- |
| **Название** | Непустое поле | `alert('Заполните обязательные поля')` |
| **Цена за ед.** | Число > 0 | `type="number"` |
| **Срок годности** | Формат YYYY-MM | `type="month"`; при сохранении вычисляется `status: 'expired'/'ok'` |
| **Просрочка** | Подсветка красным | Строка получает класс `expired-row`; иконка ⚠️ рядом со сроком |
| **Списание просрочки** | Массовое удаление | Кнопка «Списать просроченные» удаляет все записи со `status = 'expired'` |
| **Удаление** | Требует подтверждения | `confirm('Удалить препарат из базы данных?')` |

## 7. Аналитика и отчёт о выручке — `AnalyticsPage.vue` / `RevenuePage.vue`
| Поле | Ограничение | Логика UI |
| :--- | :--- | :--- |
| **Дата начала / окончания** | Обе даты обязательны | `type="date"`; порядок проверяется перед формированием отчёта |
| **Данные аналитики** | Только чтение | Все KPI-плитки и таблицы — read-only, редактирование недоступно |

## 8. Бизнес-правила системы (ограничения логики)
| Правило | Описание |
| :--- | :--- |
| Запись на будущую дату | Дата приёма должна быть не раньше следующего дня |
| Уникальность слота врача | Врач не может принимать двух пациентов в одно время |
| Собственные питомцы | Клиент записывает на приём только своих питомцев |
| Вес в допустимом диапазоне | 0.1–300.0 кг; нарушение блокирует сохранение |
| Минимум одно назначение | Медкарта не сохраняется без услуги или медикамента |
| Контроль срока годности | Просроченные медикаменты выделяются и доступны для списания |
