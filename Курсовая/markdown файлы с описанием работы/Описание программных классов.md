# Описание программных классов и методов (Backend API)

В данном разделе приведена спецификация серверных классов (контроллеров) и объектов передачи данных (DTO), обеспечивающих функциональность системы.

| Класс (Controller) | Поля в классе (DTO / Модель) | Методы (Actions) | Тип запроса |
| :--- | :--- | :--- | :--- |
| **AuthController** | `email`, `password`, `token`, `role`, `lastName`, `firstName`, `middleName`, `phone`, `address` | `login()` — аутентификация пользователя<br>`register()` — создание нового клиента<br>`changePassword()` — обновление пароля | POST<br>POST<br>PUT |
| **PetController** | `petId`, `ownerId`, `nickname`, `species`, `breed`, `birthDate`, `currentWeight`, `photoUrl` | `getByOwner()` — список питомцев клиента<br>`addPet()` — регистрация нового животного<br>`updatePet()` — редактирование данных<br>`deletePet()` — удаление карточки | GET<br>POST<br>PUT<br>DELETE |
| **BookingController**| `appId`, `petId`, `doctorId`, `specialty`, `scheduledDate`, `timeSlot`, `comment`, `status` | `getSlots()` — получение доступного времени<br>`createApp()` — первичная запись на приём<br>`updateStatus()` — подтверждение/отказ (врач) | GET<br>POST<br>PATCH |
| **VisitController** | `visitId`, `appId`, `anamnesis`, `diagnosis`, `weightOnVisit`, `serviceList`, `medList`, `totalCost` | `saveVisitCard()` — сохранение результатов приема<br>`getJournal()` — вывод истории (Журнал здоровья)<br>`getWeightData()` — массив точек для графика | POST<br>GET<br>GET |
| **AdminController** | `revenue`, `visitsCount`, `avgCheck`, `loadFactor`, `serviceId`, `medicationId`, `expiryDate` | `getStats()` — получение аналитической сводки<br>`manageServices()` — CRUD справочника услуг<br>`manageMeds()` — CRUD медикаментов<br>`purgeExpired()` — списание просрочки | GET<br>POST/PUT<br>POST/PUT<br>DELETE |

## Техническое описание полей классов

1.  **Поля аутентификации:** используются для управления сессиями и разграничения прав доступа (Клиент / Врач / Админ).
2.  **Поля параметров приема (`anamnesis`, `diagnosis`):** имеют строгую валидацию по длине (от 10 до 2000 символов) согласно требованиям интерфейса врача.
3.  **Поля справочников:** включают в себя денежный тип (`totalCost`, `price`) и дату окончания срока годности (`expiryDate`) для автоматизированного контроля склада.
4.  **Поля визуализации:** поле `weightOnVisit` в классе `VisitController` инициирует автоматическое обновление данных в массиве для отрисовки Canvas-графика.
