# Описание программных классов и методов (Backend API)

Данный раздел описывает логику взаимодействия фронтенд-интерфейса с серверной частью системы через REST API контроллеры.

## 1. AuthController (Управление доступом)
Класс отвечает за регистрацию и вход пользователей в систему.
- `doLogin(LoginDTO data)`: **POST**. Проверка email и пароля. Возвращает токен доступа и роль (Client, Doctor, Admin).
- `doRegister(RegisterDTO data)`: **POST**. Создание новой записи в таблице `owners`.
- `changePassword(PasswordDTO data)`: **PUT**. Смена пароля из профиля пользователя.

## 2. PetController (Управление картотекой)
Класс для работы с данными животных.
- `getPetsByOwner(int ownerId)`: **GET**. Получение списка питомцев для отображения в модуле «Мои питомцы».
- `addPet(PetDTO data)`: **POST**. Сохранение новой карточки (кличка, вид, порода, дата рожд., фото).
- `updatePet(int petId, PetDTO data)`: **PUT**. Редактирование данных существующего животного.
- `deletePet(int petId)`: **DELETE**. Удаление карточки (с подтверждением в модальном окне).

## 3. BookingController (Мастер записи)
Класс управления расписанием и заявками.
- `getAvailableSlots(int doctorId, Date date)`: **GET**. Возвращает список свободных 30-минутных интервалов.
- `createAppointment(AppDTO data)`: **POST**. Создание заявки со статусом `waiting`.
- `updateStatus(int appId, String status)`: **PATCH**. Изменение статуса на `confirmed` или `rejected` врачом.

## 4. VisitController (Ведение приема)
Основной рабочий класс для роли «Врач».
- `openVisit(int appId)`: **GET**. Подгрузка данных пациента в двухпанельный интерфейс приема.
- `saveMedicalCard(VisitDTO data)`: **POST**. Сохранение анамнеза, диагноза и списка назначений.
- `getHealthJournal(int petId)`: **GET**. Вывод истории визитов для модуля «Журнал здоровья».
- `getWeightHistory(int petId)`: **GET**. Получение массива данных (дата/вес) для отрисовки графика на Canvas.

## 5. AdminController (Аналитика и справочники)
- `getAnalyticsSummary()`: **GET**. Расчет KPI (выручка, ср. чек) для дашборда аналитики.
- `manageCatalog(ItemDTO data)`: **POST/PUT/DELETE**. CRUD операции для справочников Услуг и Медикаментов.
- `dischargeExpiredMeds()`: **DELETE**. Массовое списание препаратов, у которых `expiry_date < today`.
