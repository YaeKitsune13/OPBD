## Часть 1. Vue-компоненты (Frontend)

### 1.1 Экраны приложения

#### `Auth.vue` — Экран авторизации
| Элемент | Тип | Описание |
| :--- | :--- | :--- |
| **Props** | `modelValue: Boolean` | Управляет видимостью экрана (v-model из App.vue) |
| **Emits** | `update:modelValue` | Скрыть экран авторизации |
| **Emits** | `login-success(role)` | Передать роль (`client`/`doctor`/`admin`) в App.vue |
| **State** | `activeTab: ref('login')` | Переключение между формой входа и регистрации |
| **State** | `loginData: reactive` | `{ email, pass }` — поля формы входа |
| **State** | `regData: reactive` | `{ lastName, firstName, phone, email, pass }` — поля регистрации |
| **Method** | `doLogin()` | Валидация полей, определение роли по email, вызов `finalizeLogin(role)` |
| **Method** | `doRegister()` | Проверка обязательных полей, регистрация нового клиента (`role='client'`) |
| **Method** | `quickLogin(role)` | Dev-кнопки быстрого входа; вызывает `finalizeLogin(role)` |
| **Method** | `finalizeLogin(role)` | emit `login-success`, emit `update:modelValue(false)`, очистка паролей |

#### `Main.vue` — Основная оболочка
| Элемент | Тип | Описание |

## Часть 2. Backend API (контроллеры)

| Класс (Controller) | Поля в классе (DTO / Модель) | Методы (Actions) | Тип запроса |
| :--- | :--- | :--- | :--- |
| **AuthController** | `email`, `password`, `token`, `role`, `lastName`, `firstName`, `middleName`, `phone`, `address` | `login()` — аутентификация пользователя<br>`register()` — создание нового клиента<br>`changePassword()` — обновление пароля | POST<br>POST<br>PUT |
| **PetController** | `petId`, `ownerId`, `nickname`, `species`, `breed`, `birthDate`, `currentWeight`, `photoUrl` | `getByOwner()` — список питомцев клиента<br>`addPet()` — регистрация нового животного<br>`updatePet()` — редактирование данных<br>`deletePet()` — удаление карточки | GET<br>POST<br>PUT<br>DELETE |
| **BookingController** | `appId`, `petId`, `doctorId`, `specialty`, `scheduledDate`, `timeSlot`, `comment`, `status` | `getSlots()` — получение доступного времени<br>`createApp()` — первичная запись на приём<br>`updateStatus()` — подтверждение/отказ (врач) | GET<br>POST<br>PATCH |
| **VisitController** | `visitId`, `appId`, `anamnesis`, `diagnosis`, `serviceList`, `medList`, `totalCost` | `saveVisitCard()` — сохранение результатов приема<br>`getJournal()` — вывод истории (Журнал здоровья)<br>`getWeightData()` — массив точек для графика | POST<br>GET<br>GET |
| **AdminController** | `revenue`, `visitsCount`, `avgCheck`, `serviceId`, `medicationId`, `expiryDate` | `getStats()` — получение аналитической сводки<br>`manageServices()` — CRUD справочника услуг<br>`manageMeds()` — CRUD медикаментов<br>`purgeExpired()` — списание просрочки | GET<br>POST/PUT<br>POST/PUT<br>DELETE |

### Техническое описание полей классов

1. **Поля аутентификации:** используются для управления сессиями и разграничения прав доступа (Клиент / Врач / Админ). Роль хранится в JWT-токене.
2. **Поля параметров приема (`anamnesis`, `diagnosis`):** имеют строгую валидацию по длине (от 10 до 2000 символов) согласно требованиям интерфейса врача.
3. **Поля справочников:** включают в себя денежный тип (`totalCost`, `price`) и дату окончания срока годности (`expiryDate`) для автоматизированного контроля склада.
4. **Связующая таблица `visit_prescriptions`:** обеспечивает связь «многие-ко-многим» между визитом и позициями (услугами и медикаментами); поле `unit_price` фиксирует цену на момент назначения.
