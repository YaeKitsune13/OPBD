# Описание входной и выходной информации

## Входная информация

### 1. Экран авторизации — `Auth.vue`

#### 1.1 Форма «Вход»
| Поле | HTML-элемент | Формат | Обязательность | Пример |
| :--- | :--- | :--- | :--- | :--- |
| Email | `input[type=email]` | адрес электронной почты | Да | `ivanov@mail.ru` |
| Пароль | `input[type=password]` | строка, мин. 8 символов | Да | `password123` |

Роль определяется автоматически по email:
- содержит `doctor` → роль `doctor`
- содержит `admin` → роль `admin`
- остальные → роль `client`

#### 1.2 Форма «Регистрация»
| Поле | HTML-элемент | Формат | Обязательность | Пример |
| :--- | :--- | :--- | :--- | :--- |
| Фамилия | `input[type=text]` | текст | Нет | `Иванов` |
| Имя | `input[type=text]` | текст | Да | `Иван` |
| Телефон | `input[type=text]` | `+79XXXXXXXXX` | Нет | `+79001234567` |
| Email | `input[type=email]` | адрес электронной почты | Да | `ivan@mail.ru` |
| Пароль | `input[type=password]` | строка, мин. 8 символов | Нет | `mypass123` |

### 2. Карточка питомца — `PetsPage.vue` (модальное окно BaseModal)
| Поле | HTML-элемент | Формат | Обязательность | Пример |
| :--- | :--- | :--- | :--- | :--- |
| Кличка | `input[type=text]` | только буквы, 1–30 симв. | Да | `Барсик` |
| Вид животного | `<select>` | Кошка / Собака / Кролик / Другое | Нет | `Кошка` |
| Порода | `input[type=text]` | текст | Нет | `Британец` |
| Дата рождения | `input[type=date]` | DATE, не из будущего | Да | `2021-03-15` |
| Текущий вес (кг) | `input[type=number]` | DECIMAL (0.1–300.0), шаг 0.1 | Нет | `4.2` |

### 3. Запись на приём — `BookPage.vue`
| Поле | HTML-элемент | Формат | Обязательность | Пример |
| :--- | :--- | :--- | :--- | :--- |
| Питомец | `<select>` | выбор из своих питомцев | Нет | `🐱 Барсик` |
| Специализация | `<select>` | Терапевт / Хирург / Кардиолог | Нет | `Терапевт` |
| Врач | `<select>` (зависит от специализации) | ФИО из справочника | Нет | `Кузнецов А.В.` |
| Дата приёма | `input[type=date]` | DATE ≥ завтрашний день | Нет | `2026-05-10` |
| Комментарий | `<textarea>` | текст, до 1000 симв. | Нет | `Кашляет третий день` |

---

### 4. Ведение приёма — `ConductPage.vue`
| Поле | HTML-элемент | Формат | Обязательность | Пример |
| :--- | :--- | :--- | :--- | :--- |
| Питомец | `PetCombobox` (кастомный select) | объект `{ id, name, avatar, breed, owner }` | Да | Барсик — Иванов И.И. |
| Анамнез | `<textarea>` | текст, 10–2000 симв. | Да | `Владелец отмечает чихание и снижение аппетита.` |
| Диагноз | `input[type=text]` | текст | Да | `ОРВИ кошек` |
| Назначения | таблица + BaseModal | список `{ id, name, price, qty }` | Мин. 1 позиция | Первичный осмотр — 850 р. |

## Выходная информация

### 1. Экранные формы и визуализации

| Компонент | Получатель | Выходные данные |
| :--- | :--- | :--- |
| `DashboardPage.vue` | Клиент | Плитки: питомцы, записи, визиты, следующий приём; виджеты последних записей и питомцев |
| `AppointmentsPage.vue` | Клиент | Таблица: #, питомец, врач, дата/время, статус (`Ожидание` / `Подтверждено`) |
| `HistoryPage.vue` | Клиент | Карточки визитов: диагноз, назначения, анализы, рекомендации, сумма |
| `WeightPage.vue` | Клиент | Canvas-график динамики веса + таблица измерений (дата, вес, врач) |
| `TodayPage.vue` | Врач | Список пациентов дня: время, питомец, владелец, причина, статус, кнопки действия |
| `ConductPage.vue` | Врач | Итоговая сумма назначений (`computed totalCost`) в рублях в реальном времени |
| `SearchPage.vue` | Врач | Карточка найденного питомца + история завершённых визитов |
| `AnalyticsPage.vue` | Администратор | KPI-плитки; таблица популярных услуг; таблица нагрузки врачей |
| `RevenuePage.vue` | Администратор | Итоговые плитки; таблица по дням: визиты, услуги, медикаменты, итого |
| `ServicesPage.vue` | Администратор | Таблица услуг с реактивным поиском по названию/ID |
| `MedsPage.vue` | Администратор | Таблица медикаментов; просроченные строки подсвечены красным с ⚠️ |

---

### 2. Печатные формы — `PrintZone.vue`

Компонент скрыт в браузере (`display: none`) и виден только при `@media print`.
Отображается через `Teleport to="body"` и вызывается командой `window.print()`.
