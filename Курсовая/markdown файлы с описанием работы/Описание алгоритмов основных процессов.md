# Алгоритмы работы системы

## 1. Алгоритм авторизации и определения роли

1. **Начало.** Пользователь открывает приложение; `App.vue` показывает `Auth.vue` (`modelValue = true`).
2. Пользователь заполняет поля «Email» и «Пароль» и нажимает кнопку «Войти».
3. Функция `doLogin()` в `Auth.vue` проверяет: `loginData.email` и `loginData.pass` не пусты.
   - Если пусто → `alert('Введите почту и пароль')`; **конец**.
4. Определение роли по содержимому email:
   - Содержит подстроку `'doctor'` → `role = 'doctor'`
   - Содержит подстроку `'admin'` → `role = 'admin'`
   - Иначе → `role = 'client'`
5. Вызов `finalizeLogin(role)`:
   - emit `login-success(role)` → `App.vue` получает роль и передаёт её в `Main.vue` (`initialRole`).
   - emit `update:modelValue(false)` → экран `Auth.vue` скрывается.
   - Очистка паролей: `loginData.pass = ''`, `regData.pass = ''`.
6. `Main.vue` устанавливает `role.value = initialRole` и переключает `currentPage` на стартовую
   страницу роли: `{ client: 'dashboard', doctor: 'today', admin: 'analytics' }`.
7. `SideBar.vue` перерисовывает навигационное меню в соответствии с ролью.
8. **Конец.**

## 2. Алгоритм регистрации нового клиента

1. **Начало.** Пользователь нажимает вкладку «Регистрация» в `Auth.vue` (`activeTab = 'reg'`).
2. Заполняет поля: Фамилия, Имя, Телефон, Email, Пароль.
3. Нажимает «Зарегистрироваться» → вызов `doRegister()`.
4. Проверка: поля `regData.firstName` и `regData.email` не пусты.
   - Если пусто → `alert('Заполните обязательные поля')`; **конец**.
5. Вызов `finalizeLogin('client')` — новый пользователь всегда получает роль `client`.
6. **Конец.** (логика та же, что в шаге 5–7 алгоритма авторизации).

## 3. Алгоритм записи на приём

1. **Начало.** Клиент открывает страницу «Записаться» (`BookPage.vue`).
2. В форме выбирает:
   - Питомца из `<select>` (собственные питомцы).
   - Специализацию из `<select>` → `selectedSpec`.
3. После выбора специализации `computed availableDoctors` фильтрует объект `doctors` по ключу
   `selectedSpec` и разблокирует `<select>` врача.
4. Пользователь выбирает врача, дату приёма и вводит комментарий.
5. Нажимает «Отправить заявку» → `sendRequest()`:
   - Выполняет `alert('Заявка отправлена! Ожидайте подтверждения.')`.
   - emit `navigate('appointments')` → `Main.vue` переключает `currentPage = 'appointments'`.
6. В `AppointmentsPage.vue` новая запись отображается со статусом «Ожидание» (`badge-waiting`).
7. **Конец.**

## 4. Алгоритм подтверждения/отклонения заявки (врач)

1. **Начало.** Врач открывает страницу «Расписание» (`TodayPage.vue`).
2. Для записи со `status = 'waiting'` отображаются кнопки «✓» и «✕».
3. При нажатии «✓»:
   - `confirm(id)` находит элемент в `appointments.value` и устанавливает `status = 'confirmed'`.
   - Кнопки «✓»/«✕» заменяются кнопкой «Принять».
4. При нажатии «✕» (заглушка): планируется изменение статуса на `'rejected'`.
5. При нажатии «Принять» → `startVisit(app)` emit `navigate('conduct')`.
6. **Конец.**

---

## 5. Алгоритм проведения приёма (заполнение медкарты)

1. **Начало.** Врач перешёл на страницу «Ведение приёма» (`ConductPage.vue`).
2. Через `PetCombobox` выбирает питомца из базы (по умолчанию — первый в списке).
3. В поле «АНАМНЕЗ» вводит жалобы и предысторию (обязательно, 10–2000 симв.).
4. В поле «ДИАГНОЗ» вводит установленный диагноз.
5. В блоке «Назначения» нажимает «+ Добавить услугу»:
   - Открывается `BaseModal` со списком услуг из справочника.
   - Выбранная позиция добавляется в `assignments[]`.
6. `computed totalCost` пересчитывает итог: сумма `price * qty` всех строк назначений.
   Результат отображается в строке «Итого» в реальном времени.
7. При необходимости добавляет дополнительные позиции (шаги 5–6 повторяются).
8. Нажимает «Сохранить прием»:
   - emit `navigate('today')` → возврат на расписание.
   - (В продакшне: POST-запрос `VisitController.saveVisitCard()` с данными медкарты.)
9. Итоговая стоимость сохраняется в `visits.total_cost`; запись в `weight_history` обновляется.
10. **Конец.**

## 6. Алгоритм автоматического расчёта стоимости визита

1. **Начало.** Активна страница `ConductPage.vue` с непустым `assignments[]`.
2. Vue вычисляет `computed totalCost`:
   - Инициализация `sum = 0`.
   - Для каждого элемента `item` в `assignments.value`:
     `sum = sum + item.price * item.qty`.
   - Возвращает `sum`.
3. Результат немедленно отображается в поле `{{ totalCost }} руб.`.
4. При добавлении или удалении позиции Vue автоматически пересчитывает значение.
5. При нажатии «Сохранить прием» значение `totalCost` записывается в `visits.total_cost`.
6. **Конец.**

## 7. Алгоритм управления справочником услуг (администратор)

1. **Начало.** Администратор открывает страницу «Справочник услуг» (`ServicesPage.vue`).
2. `computed filteredServices` реактивно фильтрует `services[]` по `searchQuery`.
3. **Добавление:**
   - Нажать «✚ Добавить услугу» → открывается `BaseModal`.
   - Заполнить поля: название, описание, стоимость.
   - Нажать «Сохранить» → `saveService()`:
     - Проверка: `name` и `price` не пусты.
     - Генерация `id = String(services.length + 1).padStart(3, '0')`.
     - Добавление объекта в `services.value`.
     - Закрытие модального окна и сброс формы.
4. **Редактирование:** (заглушка) кнопка ✏ для будущей функциональности.
5. **Удаление:**
   - Нажать 🗑 → `deleteService(id)`:
     - `confirm('Удалить эту услугу из справочника?')`.
     - При подтверждении: `services.value = services.value.filter(s => s.id !== id)`.
6. **Конец.**

## 8. Алгоритм контроля сроков годности медикаментов (администратор)

1. **Начало.** Администратор открывает страницу «Справочник медикаментов» (`MedsPage.vue`).
2. При загрузке для каждого `m` в `meds[]`: функция `isExpired(m.expiry)` сравнивает
   `new Date(m.expiry)` с `new Date()`.
   - `true` → `m.status = 'expired'`.
   - `false` → `m.status = 'ok'`.
3. В таблице строки со `status = 'expired'` получают класс `expired-row` (красная подсветка)
   и иконку ⚠️ рядом со сроком.
4. При добавлении нового медикамента (`saveMed()`) статус вычисляется сразу через `isExpired()`.
5. **Массовое списание:**
   - Нажать «Списать просроченные».
   - `meds.value = meds.value.filter(m => m.status !== 'expired')` — все просроченные удаляются.
6. **Конец.**

## 9. Алгоритм контроля нагрузки врачей (аналитика)

1. Выборка всех активных врачей из таблицы `doctors`.
2. Для каждого врача подсчёт количества записей в статусе `confirmed` за текущий месяц.
3. Расчёт коэффициента: `(Факт. записей / Норма 160 слотов) * 100`.
4. Определение статуса для UI:
   - Более 70% → `badge-confirmed` (Высокая);
   - 40–70% → `badge-waiting` (Средняя);
   - Менее 40% → `badge-info` (Низкая).
5. Вывод данных в таблицу «Нагрузка врачей» на `AnalyticsPage.vue`.

## 10. Алгоритм обновления динамики веса

1. При сохранении медкарты визита считывается вес питомца, зафиксированный врачом.
2. Создаётся новая запись в таблице `weight_history` (`pet_id`, вес, `visit_id`, `measured_at`).
3. Обновляется поле `current_weight` в основной таблице `pets`.
4. При открытии страницы «Динамика веса» массив `weightData` в `WeightPage.vue` заполняется
   точками из `weight_history`.
5. Функция `drawChart()` отрисовывает линейный график через Canvas API с подписями по оси X.
6. При изменении размера окна браузера `resize`-слушатель вызывает `drawChart()` повторно.
