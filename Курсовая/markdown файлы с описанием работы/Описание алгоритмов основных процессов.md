# Алгоритмы работы системы

## 1. Алгоритм автоматического расчета стоимости визита
1. **Начало.**
2. Инициализация переменной `TotalSum = 0`.
3. Получение списка ID выбранных услуг из таблицы назначений.
4. Для каждой услуги: `TotalSum = TotalSum + Service.Price`.
5. Получение списка ID медикаментов и их количества (`Qty`).
6. Для каждого медикамента: `TotalSum = TotalSum + (Medication.Price * Qty)`.
7. Отображение `TotalSum` в интерфейсе врача в реальном времени.
8. При сохранении: запись итоговой суммы в поле `visits.total_cost`.
9. **Конец.**

## 2. Алгоритм контроля нагрузки врачей (Аналитика)
1. Выборка всех активных врачей из таблицы `doctors`.
2. Для каждого врача подсчет количества записей в статусе `confirmed` за месяц.
3. Расчет коэффициента: `(Факт. записей / Норма в 160 слотов) * 100`.
4. Определение статуса для UI:
   - Более 70% — статус `badge-confirmed` (Высокая);
   - 40-70% — статус `badge-waiting` (Средняя);
   - Менее 40% — статус `badge-info` (Низкая).
5. Вывод данных в таблицу модуля «Нагрузка врачей».

## 3. Алгоритм обновления динамики веса
1. При сохранении медкарты визита считывается поле «Вес на приеме».
2. Создается новая запись в таблице `weight_history` (pet_id, вес, дата_визита).
3. Обновляется поле `current_weight` в основной таблице `pets`.
4. При открытии страницы «Динамика веса» данные из `weight_history` передаются в скрипт отрисовки графика (Canvas).
