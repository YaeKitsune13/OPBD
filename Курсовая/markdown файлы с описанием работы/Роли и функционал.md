# Роли и функциональные модули системы

Система поддерживает три роли пользователей. Роль определяется при входе и управляет отображением
навигационного меню в компоненте `SideBar.vue`, а также набором доступных страниц в `Main.vue`.

---

## 1. Роль: Клиент (Владелец животного)

Начальная страница после входа — **Обзор** (`currentPage = 'dashboard'`).

### Модуль «Обзор» — `DashboardPage.vue`
Сводная панель управления клиента.
- Приветствие с именем и текущей датой.
- Четыре информационных плитки (stats-grid): количество питомцев, количество активных записей,
  общее число визитов, дата и время ближайшего приёма.
- Виджет «Последние записи» со статусом и кнопкой перехода на страницу записей.
- Виджет «Мои питомцы» с краткой информацией и кнопкой перехода к картотеке.
- Кнопка «✚ Записаться» — прямой переход на страницу записи.
- **Результат:** клиент видит актуальную сводку, не заходя в подразделы.

### Модуль «Мои питомцы» — `PetsPage.vue`
Управление картотекой животных клиента.
- Сетка карточек (pet-grid) с аватаром-эмодзи, кличкой, видом, породой и весом.
- Кнопка «✚ Добавить питомца» — открывает `BaseModal` с формой: кличка, вид, порода, дата рождения, вес.
- Кнопка «✏ Изменить» (заглушка) и «🗑 Удалить» — открывает `BaseModal` с подтверждением.
- Карточка-заглушка «+ Добавить питомца» в конце сетки.
- **Результат:** владелец управляет перечнем своих животных; изменения хранятся реактивно.

### Модуль «Записи на приём» — `AppointmentsPage.vue`
Таблица активных заявок клиента.
- Столбцы: №, Питомец, Врач · Специальность, Дата и время, Статус, Действие.
- Статусы: «Ожидание» (`badge-waiting`) и «Подтверждено» (`badge-confirmed`).
- Кнопка «Отменить» для каждой записи.
- Кнопка «✚ Новая запись» — переход на `BookPage`.
- **Результат:** клиент отслеживает актуальный статус своих заявок.

### Модуль «Записаться» — `BookPage.vue`
Интерактивная форма создания новой записи на приём.
- Поля формы: выбор питомца, выбор специализации, выбор врача (фильтруется по специализации),
  дата приёма, комментарий.
- Список врачей по специализации: терапевт, хирург, кардиолог, невролог, дерматолог.
- После отправки — автоматический переход на страницу «Записи» (`AppointmentsPage`).
- **Результат:** создаётся новая заявка в статусе «Ожидание».

### Модуль «Журнал здоровья» — `HistoryPage.vue`
Электронная медицинская карта питомца.
- Выбор питомца через `<select>`.
- Список карточек визитов: дата, врач, статус «Завершён», диагноз, детали назначений,
  результаты анализов (если есть), рекомендации, итоговая сумма.
- Кнопка «🖨️ Печать» для всего журнала и отдельная кнопка для каждого визита.
  При нажатии передаёт данные в `PrintZone.vue` через emit `print`, затем вызывает `window.print()`.
- **Результат:** клиент просматривает полную историю лечения и распечатывает выписку.

### Модуль «Динамика веса» — `WeightPage.vue`
Визуализация изменения веса питомца.
- Выбор питомца через `<select>`.
- Canvas-график (`<canvas ref="canvasRef">`) с отрисовкой через Canvas API; пересчёт
  при изменении размера окна (`resize`-listener).
- Таблица взвешиваний: дата, вес (кг), врач.
- **Результат:** владелец видит динамику здоровья питомца в виде графика.

## 2. Роль: Врач

Начальная страница после входа — **Расписание** (`currentPage = 'today'`).

### Модуль «Расписание» — `TodayPage.vue`
Список пациентов врача на текущий день.
- Для каждой записи: время, питомец и эмодзи, владелец, порода, причина визита, статус.
- Подтверждённые записи: кнопка «Принять» → переход на `ConductPage`.
- Записи в ожидании: кнопки «✓» (подтвердить) и «✕» (отклонить) с изменением статуса реактивно.
- Счётчик записей на день (`badge-info`).
- **Результат:** врач управляет своим расписанием и запускает приём.

### Модуль «Ведение приёма» — `ConductPage.vue`
Рабочее место врача во время осмотра.
- Заголовок с выбором питомца через `PetCombobox.vue` (кастомный выпадающий список с поиском).
- **Левая карточка «Медкарта»:**
  - Поле «АНАМНЕЗ» (`<textarea>`) — жалобы и предыстория.
  - Поле «ДИАГНОЗ» (`<input>`) — установленный диагноз.
- **Правая карточка «Назначения»:**
  - Таблица добавленных услуг/медикаментов: наименование, цена, кнопка удаления (✕).
  - Кнопка «+ Добавить услугу» — открывает `BaseModal` с выбором из справочника.
  - Строка итога с автоматически вычисляемой суммой (`computed totalCost`).
- Кнопки footer: «Отмена» → `TodayPage`, «Сохранить прием» → `TodayPage`.
- **Результат:** заполненная медкарта визита с назначениями и итоговой стоимостью.

### Модуль «Поиск пациента» — `SearchPage.vue`
Глобальный поиск по базе клиники.
- Строка поиска по кличке питомца или ФИО владельца; поиск активируется при длине > 2 символов.
- После поиска: карточка питомца (вид, порода, вес, возраст, ФИО и телефон владельца)
  и список завершённых визитов с диагнозом.
- Пустое состояние с иконкой 🔍 пока запрос не введён.
- **Результат:** врач быстро находит историю болезни любого животного.

## 3. Роль: Администратор

Начальная страница после входа — **Аналитика** (`currentPage = 'analytics'`).

### Модуль «Аналитическая сводка» — `AnalyticsPage.vue`
Дашборд ключевых показателей (KPI) клиники за текущий месяц.
- Четыре плитки: визитов за месяц, выручка (руб.), активных клиентов, средний чек.
- Таблица «Популярные услуги»: наименование, количество оказаний, выручка.
- Таблица «Нагрузка врачей»: ФИО, количество приёмов, статус нагрузки (Высокая/Средняя).
- **Результат:** администратор оценивает эффективность работы клиники.

### Модуль «Выручка» — `RevenuePage.vue`
Финансовый отчёт за выбранный период.
- Фильтр по датам: два поля `<input type="date">` и кнопка «Сформировать».
- Плитки итогов: суммарная выручка, выручка за услуги (с % от общей).
- Таблица по дням: дата, количество визитов, услуги (руб.), медикаменты (руб.), итого (руб.).
- **Результат:** детализированный финансовый отчёт за период.

### Модуль «Услуги» — `ServicesPage.vue`
Справочник процедур клиники (CRUD).
- Строка поиска — реактивная фильтрация по названию или ID.
- Таблица: #, наименование (с описанием), стоимость, действия (✏ редактировать, 🗑 удалить).
- Кнопка «✚ Добавить услугу» — открывает `BaseModal` с полями: название, описание, стоимость.
- Удаление — нативный `confirm()`.
- **Результат:** актуальный прайс-лист услуг клиники.

### Модуль «Медикаменты» — `MedsPage.vue`
Управление аптечным складом (CRUD).
- Строка поиска — реактивная фильтрация по названию или артикулу.
- Таблица: #, наименование (с описанием), цена за ед., срок годности, действия.
- Просроченные позиции выделяются красным (класс `expired-row`), рядом — иконка ⚠️.
- Кнопка «✚ Добавить медикамент» — открывает `BaseModal`: название, описание, цена, срок годности.
- Кнопка «Списать просроченные» — массовое удаление позиций с истёкшим сроком.
- **Результат:** актуальный склад медикаментов с контролем годности.
