# Роли и функционал системы ветеринарной клиники

## Роли

| Роль | Основной функционал | Аналитика и отчётность |
| :--- | :--- | :--- |
| **Клиент** | Регистрация, добавление питомцев, запись на приём. | Просмотр графика веса и истории оплат. |
| **Врач** | Просмотр записей на день, заполнение диагноза, регистрация питомца. | Просмотр истории болезней и статусов заявок. |

---

## Модуль Клиента

| Функция | Описание |
| :--- | :--- |
| **Регистрация / Профиль** | Создание аккаунта и изменение контактных данных. |
| **Картотека питомцев** | Добавление и редактирование карточки питомца. |
| **Запись на приём** | Выбор питомца, врача и удобного времени. |
| **Журнал здоровья** | Просмотр прошедших приёмов, диагнозов и назначений. |
| **Личная статистика** | Просмотр графика веса и частоты посещений питомца. |
| **Статусы заявок** | Просмотр состояния записей: Ожидание / Подтверждено / Отклонено. |

---

## Модуль Врача

| Функция | Описание |
| :--- | :--- |
| **Панель «Сегодня»** | Просмотр своих записей на текущий день. |
| **Ведение приёма** | Изменение статуса заявки, заполнение анамнеза и диагноза. |
| **Назначение лечения** | Выбор процедур и лекарств, расчёт стоимости приёма. |
| **Поиск пациента** | Просмотр истории болезней любого питомца в базе. |
| **Быстрая регистрация** | Добавление нового питомца прямо во время приёма. |

---

## Системные требования

| Правило | Описание |
| :--- | :--- |
| **Валидация дат** | Нельзя записаться на прошедшую дату. |
| **Без пересечений** | Врач не может вести двух пациентов одновременно. |
| **Защита данных** | Клиент видит только своих питомцев; врач — только медицинские данные. |
