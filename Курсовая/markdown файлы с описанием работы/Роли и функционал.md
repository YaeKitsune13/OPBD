# Роли и функционал системы ветеринарной клиники

| Роль | Основной функционал | Аналитика и отчетность |
| :--- | :--- | :--- |
| **Пользователь (Клиент)** | • Регистрация в системе и добавление питомцев.<br>• Создание заявок на прием.<br>• Просмотр электронной медкарты питомца. | • Мониторинг состояния (статистика здоровья/веса).<br>• История платежей за лечение. |
| **Врач** | • Управление списком приемов на день.<br>• Заполнение протокола осмотра и постановка диагноза.<br>• Регистрация новых питомцев (если клиент пришел впервые). | • Доступ к истории болезни (архив лечений).<br>• Контроль статусов выполнения заявок. |
| **Админ (Владелец)** | • Управление персоналом (добавление/блокировка врачей).<br>• Модерация справочников (услуги, цены). | • Финансовая аналитика (доходы vs расходы на зарплаты).<br>• Анонимная статистика по посещаемости клиники. |

# Подробный функционал курсового проекта: "Информационная система ветклиники"

## 1. Модуль Пользователя (Клиент)
| Функция | Описание |
| :--- | :--- |
| **Регистрация / Профиль** | Создание аккаунта, управление контактными данными (телефон, почта). |
| **Картотека питомцев** | Добавление/редактирование профилей животных (кличка, вид, порода, возраст, фото). |
| **Запись на прием** | Создание заявки: выбор питомца, специализации врача и желаемой даты/времени. |
| **Журнал здоровья** | Список всех завершенных приемов с назначениями и диагнозами (электронная медкарта). |
| **Личная статистика** | Визуализация данных (например, график веса питомца или частота посещений). |
| **Статусы заявок** | Просмотр состояния текущих записей (Ожидание / Подтверждено / Отклонено). |

## 2. Модуль Врача (Рабочее место)
| Функция | Описание |
| :--- | :--- |
| **Панель «Сегодня»** | Список всех записей, назначенных именно на текущего врача на текущий день. |
| **Ведение приема** | Смена статуса заявки ("В процессе"), заполнение текстового поля "Анамнез" и "Диагноз". |
| **Назначение лечения** | Выбор процедур или лекарств из справочника, расчет итоговой стоимости приема. |
| **Поиск пациента** | Доступ к истории болезней любых питомцев по базе (для просмотра предыдущих записей коллег). |
| **Быстрая регистрация** | Возможность завести карточку нового питомца прямо во время экстренного приема. |

## 3. Модуль Администратора (Управление)
| Функция | Описание |
| :--- | :--- |
| **Штат сотрудников** | Создание учетных записей для новых врачей, управление их ролями и доступами. |
| **Финансовый отчет** | Просмотр общей суммы выручки за месяц (на основе стоимости завершенных приемов). |
| **Расходы на ЗП** | Учет суммы выплат врачам (фиксированный оклад или процент от приемов). |
| **Справочник услуг** | Управление прайс-листом (названия услуг и их цена). |
| **Анонимная аналитика** | Сводные графики прибыли, самых популярных услуг и нагрузки по дням недели. |

## 4. Системные требования (Логика)
- [x] **Валидация дат:** Нельзя записаться на "вчера".
- [x] **Контроль пересечений:** Врач не может принять двоих одновременно.
- [x] **Безопасность данных:** Клиент видит только своих питомцев, врач — только медицинские данные, админ — только финансовые.
