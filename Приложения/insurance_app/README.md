# Учёт страховой компании — PyQt6 + MySQL

Десктопное приложение на PyQt6 для работы с базой данных страховой компании:
добавление / редактирование / удаление записей по 4 таблицам + готовые отчёты.

## Структура проекта

```
insurance_app/
├── sql/schema.sql       # создание БД, таблиц, триггеров и тестовых данных
├── main.py              # точка входа
├── db_manager.py        # подключение к MySQL и CRUD-запросы
├── table_specs.py       # описание полей каждой таблицы (на основе него строится UI)
├── connection_dialog.py # окно ввода параметров подключения
├── crud_tab.py          # универсальная вкладка "таблица + форма"
├── reports_tab.py       # вкладка с 5 готовыми отчётами
├── main_window.py        # главное окно с вкладками
└── requirements.txt
```

## 1. Поднять базу данных

Нужен установленный MySQL (или MariaDB) сервер. Выполните:

```bash
mysql -u root -p < sql/schema.sql
```

Скрипт создаст базу `insurance`, 4 таблицы, тестовые данные и **триггеры**.

### Зачем нужны триггеры

В таблице `employees` есть поле `count_policyholders`, которое нигде не
заполняется вручную — оно должно поддерживаться автоматически. Поэтому
добавлены 3 триггера на таблице `policyholders`:

- `trg_policyholders_after_insert` — при добавлении полиса увеличивает
  счётчик у ответственного сотрудника;
- `trg_policyholders_after_update` — при смене ответственного сотрудника
  у полиса уменьшает счётчик у старого и увеличивает у нового;
- `trg_policyholders_after_delete` — при удалении полиса уменьшает счётчик.

Правила «выплаты не больше премии» и «первый случай не раньше 6 дней после
заключения» **не сделаны блокирующими триггерами** намеренно: тестовые
данные содержат полис `pl00000002`, который специально нарушает оба этих
правила, чтобы отчёты № 1 и № 2 могли найти нарушение. Блокирующий триггер
просто не дал бы загрузить тестовые данные. Эти правила реализованы как
отчёты (вкладка «Отчёты» в приложении).

## 2. Установить зависимости Python

```bash
pip install -r requirements.txt
```

## 3. Запустить приложение

```bash
python main.py
```

При старте появится окно подключения — укажите хост, порт, пользователя,
пароль и имя базы (`insurance` по умолчанию).

## Возможности приложения

- 4 вкладки CRUD: «Виды страхования», «Сотрудники», «Страхователи (полисы)»,
  «Страховые случаи» — добавление, изменение, удаление записей, с выпадающими
  списками для внешних ключей (вид страхования, сотрудник, номер полиса).
- Вкладка «Отчёты» — 5 запросов из задания (выплаты > премии, ранние
  страховые случаи, страхователи по видам страхования, сотрудники без
  договоров за сегодня, расхождение стоимости полиса с годовой стоимостью).
- Понятные сообщения об ошибках при нарушении внешних ключей, дублировании
  ключа или срабатывании триггера.

## Возможные доработки

- Хранение пароля от БД через переменные окружения вместо ввода каждый раз.
- Поиск/фильтрация в таблицах.
- Экспорт отчётов в Excel/PDF.
